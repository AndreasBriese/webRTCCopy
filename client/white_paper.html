Good news! WebRTC already mandates encryption over DTLS!
http://chimera.labs.oreilly.com/books/1230000000545/ch18.html#_secure_communication_with_dtls

Bad news: there is no authentication scheme. This website gets around that, not by requiring 3rd party authorization,
but by negotiating an OTR encrypted tunnel between users. While this is potentially overkill, OTR does offer an added level of security.

Details:
1) We use OTR's SMP in order to verify password both parties entered is the same
2) Files are encrypted using RC4Drop with (the password negotiated during OTR setup + a random per-chunk nonce sent over from receiving end sent over OTR channel)
3) Once files are received a hash is sent back to the other party over the OTR channel in order to verify file integrity
